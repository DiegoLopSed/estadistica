Puertos Digitales — Conceptos Básicos

  • Función: Interactuar con el mundo exterior mediante señales binarias (0 y 1)
  • Configuración: Entrada o salida según necesidad
  • Niveles lógicos: 0V (bajo) y Vcc (alto, típicamente 3.3V o 5V)
5 / 26

Puertos Digitales — Características

  • Agrupación en puertos: 8, 16 o 32 bits
  • Control individual: Cada pin configurable independientemente
  • Pull-up/Pull-down: Resistencias internas para estados estables
  • Limitación de corriente: 20-40 mA por pin típicamente
  • Interrupciones: Respuesta rápida a eventos externos
6 / 26

Puertos Digitales — Aplicaciones

  • Control de LEDs y displays de 7 segmentos
  • Lectura de botones y switches
  • Activación de relés
  • Control de motores paso a paso
  • Monitoreo de eventos discretos
7 / 26

Puertos Analógicos — Conversión ADC

  • Función: Transformar señales continuas en valores digitales
  • Resolución: 10, 12 o 16 bits (determina precisión)
  • Velocidad de muestreo: Conversiones por segundo
  • Rango de voltaje: Normalmente 0V a Vcc
  • Referencias: Internas o externas para mayor precisión
8 / 26

Puertos Analógicos — DAC y PWM

  • DAC: Genera voltajes variables desde valores digitales
  • PWM (alternativa): Modulación por ancho de pulso para simular salidas analógicas
  • Aplicación PWM: Control de velocidad de motores, atenuación de LEDs
9 / 26

Puertos Analógicos — Aplicaciones

  • Lectura de sensores: temperatura, humedad, luz, presión
  • Acelerómetros y giróscopos
  • Control de servomotores
  • Atenuación de LEDs
  • Generación de señales de audio
10 / 26

Temporizadores — Conceptos

  • Definición: Módulos de hardware especializados que operan independientemente del procesador
  • Función principal: Mediciones precisas de tiempo y generación de eventos temporales
  • Ventaja: No consumen recursos del procesador principal
11 / 26

Temporizadores — Tipos

  • Básicos: Contadores que incrementan con cada pulso de reloj
  • De captura: Registran valor del contador al detectar evento externo
  • De comparación: Generan eventos cuando el contador alcanza un valor predeterminado
  • Watchdog: Resetean el sistema si no son reiniciados (previenen bloqueos)
12 / 26

Temporizadores — Características

  • Múltiples modos de operación
  • Prescalers para dividir frecuencia de reloj
  • Capacidad de generar interrupciones
  • Operación en modos de bajo consumo
  • Precisión dependiente de la frecuencia del reloj del sistema
13 / 26

Temporizadores — Aplicaciones

  • Generación de señales PWM para control de motores
  • Medición de frecuencias y períodos
  • Base de tiempo para sistemas operativos en tiempo real
  • Generación de retardos precisos
  • Decodificación de protocolos de comunicación
14 / 26

Interfaces de Comunicación — Introducción

  • Propósito: Intercambio de datos entre sistemas embebidos y otros dispositivos
  • Tipos: UART, SPI, I²C, CAN, USB, Ethernet, Wi-Fi
  • Criterios de selección: Velocidad, distancia, número de dispositivos, confiabilidad
15 / 26

UART — Universal Asynchronous Receiver-Transmitter

  • Tipo: Comunicación serial asíncrona
  • Líneas: TX (transmisión) y RX (recepción)
  • Velocidades: 9600, 115200 bps, etc.
  • Características: Bits de inicio, parada y paridad
  • Topología: Punto a punto
  • Ventaja: Simplicidad de implementación
16 / 26

SPI — Serial Peripheral Interface

  • Tipo: Comunicación serial síncrona de alta velocidad
  • Líneas (4):
    • MOSI: Master Out Slave In
    • MISO: Master In Slave Out
    • SCK: Serial Clock (generada por maestro)
    • SS/CS: Slave Select / Chip Select
  • Ventajas: Velocidad alta, soporta múltiples esclavos
  • Aplicaciones: Memorias flash, displays, conversores ADC externos
17 / 26

I²C — Inter-Integrated Circuit

  • Tipo: Protocolo de bus compartido
  • Líneas (2):
    • SDA: Serial Data (bidireccional)
    • SCL: Serial Clock
  • Direccionamiento: Cada dispositivo tiene dirección única
  • Ventaja: Solo dos líneas para múltiples dispositivos
  • Aplicaciones: Sensores, memorias EEPROM, expansores de E/S
18 / 26

CAN — Controller Area Network

  • Diseño: Aplicaciones automotrices e industriales
  • Características:
    • Alta confiabilidad
    • Detección y corrección de errores
    • Priorización de mensajes
    • Operación en ambientes con ruido electromagnético
  • Aplicaciones: Sistemas de control vehicular, automatización industrial
19 / 26

Otras Interfaces — USB, Ethernet, Wi-Fi

  • USB: Conexión estándar a computadoras, función como dispositivo o host
  • Ethernet: Conectividad de red cableada
  • Wi-Fi: Conectividad inalámbrica
  • Aplicaciones: IoT, monitoreo remoto, sistemas en red

Integración de Componentes — Ejemplo Práctico

Sistema de control de temperatura:

  • Puerto analógico: Lee sensor de temperatura (ADC)
  • Temporizador: Genera señal PWM para controlar ventilador
  • Puerto digital: Activa relé del sistema de calefacción
  • UART: Envía datos a computadora para monitoreo
  • I²C: Lee configuración desde memoria EEPROM
22 / 26

Consideraciones de Diseño

  • Selección de interface: Según velocidad, distancia y número de dispositivos
  • Gestión de interrupciones: Optimizar tiempos de respuesta
  • Consumo de energía: Uso eficiente de temporizadores y modos de bajo consumo
  • Robustez: Protección contra ruido y errores de comunicación
23 / 26

Conclusiones (1/2)

  • Los puertos digitales proporcionan la base para interacciones binarias esenciales
  • Los puertos analógicos extienden las capacidades al dominio de señales continuas
  • Ambos tipos de puertos son fundamentales para la interacción con el entorno físico
24 / 26

Conclusiones (2/2)

  • Los temporizadores son críticos para sincronización precisa y determinismo temporal
  • Las interfaces de comunicación permiten interconexión y colaboración entre sistemas
  • La selección apropiada de componentes es clave para el éxito del diseño embebido
