1) Identificación del “cerebro”

Dispositivo:
Game Boy Color (GBC)
Modelo de procesador:
Sharp LR35902 (CPU híbrida basada en Intel 8080 y Zilog Z80)
Frecuencia:
~8.388 MHz (doble velocidad respecto al Game Boy original)
Arquitectura:
8 bits
Filosofía de instrucciones:
CISC (Complex Instruction Set Computer)
Justificación:
  • Tiene un conjunto amplio de instrucciones, incluyendo operaciones complejas.
  • Permite instrucciones de diferentes tamaños y ciclos de ejecución variables.
  • Integra características heredadas de procesadores como el Z80, típicamente CISC.
  • Incluye instrucciones que realizan múltiples operaciones en un solo ciclo (ej. manipulación directa de memoria).
  • Aunque es simplificado comparado con CPUs de escritorio, no cumple con la filosofía RISC.

2) Mapa de Memoria

El Game Boy Color utiliza un espacio de direcciones de 16 bits, lo que permite acceder hasta 64 KB de memoria.
Dirección Tamaño Descripción
0000–3FFF16 KBROM del cartucho (banco fijo)
4000–7FFF16 KBROM con banco intercambiable
8000–9FFF8 KBVRAM (memoria de video)
A000–BFFF8 KBRAM externa (cartucho)
C000–CFFF4 KBRAM interna
D000–DFFF4 KBRAM interna con banco intercambiable
E000–FDFF7.5 KBEco de RAM (mirror)
FE00–FE9F160 bytesOAM (atributos de sprites)
FEA0–FEFF—Área no usable
FF00–FF7F128 bytesRegistros de I/O
FF80–FFFE127 bytesHRAM (RAM de alta velocidad)
FFFF1 byteRegistro de interrupciones
Notas importantes:
  • La VRAM permite trabajar con gráficos en color (hasta 56 colores simultáneos).
  • La memoria es limitada, por lo que se usan técnicas como bank switching.
  • El GBC mejora la RAM respecto al Game Boy original (~32 KB total interna).

3) Análisis de eficiencia

Fortalezas
  • Bajo consumo energético: diseñado para funcionar con baterías, optimiza cada ciclo de CPU.
  • Arquitectura simple pero efectiva: permite ejecución rápida sin hardware complejo.
  • Uso de bank switching: permite superar la limitación de 64 KB direccionables.
  • Integración CPU + GPU básica: reduce overhead y mejora eficiencia en gráficos 2D.
  • Optimización para juegos: el hardware está especializado para tareas específicas.
Limitaciones
  • Velocidad limitada: 8 MHz es bajo comparado con consolas de sobremesa.
  • Memoria restringida: dependencia de cartuchos con memoria adicional.
  • Sin multitarea real: solo ejecuta un proceso principal (foreground).
  • Sin pipeline avanzado ni paralelismo: muy básico frente a arquitecturas modernas.
Evaluación general de eficiencia
Factor Evaluación
Consumo energético5/5

Conclusión: El Game Boy Color es altamente eficiente para su propósito: ejecutar videojuegos portátiles con bajo consumo. Su arquitectura CISC simplificada permite flexibilidad, pero sacrifica rendimiento y escalabilidad frente a sistemas modernos.

Conclusión general

El Game Boy Color representa un diseño equilibrado entre limitaciones físicas (batería, tamaño) y capacidad de procesamiento, donde su CPU CISC simplificada y su manejo eficiente de memoria le permiten ofrecer una experiencia sólida dentro de un hardware muy restringido.
